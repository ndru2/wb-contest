## Симуляция данных

1. ```build_network``` строит граф с заданным количеством складов, хабов и пвз
2. ```simulate``` создает симуляцию заказов, принимая на вход количество шагов. На каждом шаге происходит генерация заказов и постепенное выполнение заказов в очереди.

## Алгоритм

1. Создание графа
2. Пошаговое выполнение симуляции

    2.1. Для каждого шага генерируется количество заказов от *min_orders_per_step* до *min_orders_per_step*
    
    2.2. Выбираются начальная вершина из подмножества вершин складов и конечная вершина из подмножества пвз так, чтобы между ними гарантированно был путь

    2.3. Добавить заказы в очереди

    2.4. Для каждого узла выполняется *capacity* заказов из очереди, очередь оновляется. Для заказов не достигших финальной точки выполняется переход в очередь к следующему узлу, либо ```reroute``` с обновленными весами и маскировкой вершины.

    2.5. Пересчет весов

    2.6. Добавление записи

3. Сохранение в csv
